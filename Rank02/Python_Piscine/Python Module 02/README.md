# Python Module 02: Exception Handling

This module focuses on writing robust Python code by properly anticipating, capturing, and managing runtime errors using exceptions.

## Exercises

| Exercise | File(s) | Description |
| :--- | :--- | :--- |
| **ex00** | `ft_first_exception.py` | Understanding `try` and `except` blocks to catch basic built-in errors. |
| **ex01** | `ft_different_errors.py` | Managing multiple exception types and handling them accordingly. |
| **ex02** | `ft_custom_errors.py` | Creating user-defined Exception classes for application-specific error conditions. |
| **ex03** | `ft_finally_block.py` | Ensuring specific code executes regardless of success or failure using the `finally` keyword. |
| **ex04** | `ft_raise_errors.py` | Actively throwing exceptions (`raise`) when invalid conditions are met in your logic. |
| **ex05** | `ft_garden_management.py` | A comprehensive exercise utilizing all acquired exception-handling techniques in a coherent garden simulation scenario. |
