# Python Module 03: Games and Data Systems

This module shifts the focus to more advanced architectural concepts, introducing game-related components, tracking systems, and handling continuous data streams.

## Exercises

| Exercise | File(s) | Description |
| :--- | :--- | :--- |
| **ex00** | `ft_command_quest.py` | Processing user input and acting upon a sequence of commands. |
| **ex01** | `ft_score_analytics.py` | Complex calculations and state tracking to manage a scoring system over time. |
| **ex02** | `ft_coordinate_system.py` | Implementing structures to handle 2D or 3D positions and movement logic. |
| **ex03** | `ft_achievement_tracker.py` | Building an event-driven system to unlock and store distinct accomplishments. |
| **ex04** | `ft_inventory_system.py` | Managing collections, dictionaries, or lists to hold, add, and drop items interactively. |
| **ex05** | `ft_data_stream.py` | Handling mock real-time data ingestion and processing the flow logically. |
| **ex06** | `ft_analytics_dashboard.py` | Consolidating data across the various systems to output a comprehensive overview dashboard. |
